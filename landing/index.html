How it works

Your agent does the work.
You get the report.

perceptdot sits between your AI agent and your external services. No dashboards to check. No manual analysis. Just answers.

Step 01
🔌

Install @perceptdot/core

Run npx -y @perceptdot/core — your agent's app store. It scans your project and recommends the right MCP servers in seconds.

Step 02
👁️

Agent reads live data

Your agent calls perceptdot tools to pull GA4 traffic, Vercel deployment status, and performance metrics in real time.

Step 03
📊

ROI report delivered

perceptdot calculates what the agent's work actually saved — time, cost, tokens — and delivers a clear summary to you.

Install

Up and running in 60 seconds

Start with @perceptdot/core — your agent's app store entry point. It discovers what your project needs, then you add individual servers.

Start Here — Install Core First
npx -y @perceptdot/core
Auto-discovers MCP servers for your project. No config needed.

2 — Install individual servers (optional)

Terminal 
npm install -g @perceptdot/ga4 @perceptdot/vercel @perceptdot/sentry @perceptdot/github

3 — Add to Claude Code config

~/.claude/claude_desktop_config.json 
{
  "mcpServers": {
    "perceptdot-core": {
      "command": "npx",
      "args": ["-y", "@perceptdot/core"]
    },
    "perceptdot-ga4": {
      "command": "npx",
      "args": ["@perceptdot/ga4"]
    },
    "perceptdot-vercel": {
      "command": "npx",
      "args": ["@perceptdot/vercel"]
    },
    "perceptdot-sentry": {
      "command": "npx",
      "args": ["@perceptdot/sentry"]
    },
    "perceptdot-github": {
      "command": "npx",
      "args": ["@perceptdot/github"]
    }
  }
}
