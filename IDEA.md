# AI Agent Observability Platform — 아이디어 기록

> 최초 기록: 2026-03-19
> 사주 프로젝트와 **별개** 프로젝트

---

## 핵심 아이디어

**고객 = AI 에이전트**

AI 에이전트(Claude Code, Cursor 등)가 개발 작업 중 외부 서비스(GA4, EC2, DB, Vercel 등)에 직접 접근할 수 있게 해주는 통합 플랫폼.

에이전트가 자신의 주인(개발자/CEO)에게:
> "이 도구 도입 후 토큰 47% 절감됨, 작업 시간 52% 단축됨"

을 **수치로 자동 보고**할 수 있으면 셀프 도입 설득이 가능해짐.

---

## 문제 정의

### AI 에이전트가 지금 못 하는 것들

| 문제 | 현재 상황 | 있었으면? |
|------|-----------|-----------|
| GA4 데이터 읽기 | 사람이 대시보드 봐야 함 | 에이전트가 직접 이벤트 수집 확인 |
| EC2 로그 감지 | 사람이 "이상한데요?" 해야 알게 됨 | 에러 발생 즉시 에이전트가 인지 |
| Vercel 배포 상태 | 배포 후 "됐겠지?" 수준 | 배포 완료/실패 자동 확인 |
| DB 데이터 조회 | SSH 들어가서 직접 쿼리 | 에이전트가 직접 SELECT |
| E2E 테스트 실패 원인 | 텍스트 로그만으로 추측 | 스크린샷/비디오 자동 첨부 |
| 사용자 행동 파악 | 사람이 설명해줘야 함 | 세션 녹화로 재현 |

### 공통 패턴
에이전트가 **보지 못하는 것들** 때문에 사람이 개입해야 함 → 토큰 낭비 + 시간 낭비

---

## 가치 제안

> **"에이전트가 눈을 갖게 되는 플랫폼"**

1. 외부 서비스 연결 (MCP 서버 형태)
2. 에이전트가 직접 데이터 읽고 판단
3. 토큰/시간 절감량 자동 측정
4. 주인에게 ROI 보고서 자동 생성
