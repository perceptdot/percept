# 일일 보고 — 2026-04-04

## 세션 1: Cloudflare Pages Git 연동 설정 (데스크탑 Claude)

### 작업 개요
- **목표**: perceptdot landing page 자동 배포 설정
- **문제**: Cloudflare Pages "git 연결 없음" → GitHub Actions로 배포 시도 → API 토큰 권한 부족
- **최종 해결**: GitHub과 Cloudflare Pages 직접 연동 (자동 배포 활성화)

### 상세 과정

#### Phase 1: GitHub Actions 배포 시도 (실패)
```
1. .github/workflows/deploy-landing.yml 작성
2. CLOUDFLARE_API_TOKEN, CLOUDFLARE_ACCOUNT_ID GitHub Secrets 설정
3. 첫 배포 실행 → wrangler CLI 버전 문제 (deprecated)
4. wrangler@latest로 업그레이드
5. 배포 재시도 → "project not found" (경로 오류)
6. 경로 수정 (perceptdot/landing → landing)
7. 배포 재시도 → "Authentication error [code: 10000]"
8. 새 API 토큰 생성 (Cloudflare Pages:Edit) → Keychain 저장
9. GitHub Secrets 업데이트
10. 배포 재시도 → 동일 권한 오류 (User Details Read 부족)
```

**근본 원인**: 
- CF Pages 프로젝트가 "manual deployment" 타입으로 생성됨
- Git 연동 옵션이 없음 → 수동 배포만 지원

#### Phase 2: GitHub + CF Pages 직접 연동 (성공)
```
1. Cloudflare 대시보드 → Pages → "Create a project"
2. "Connect GitHub" 선택
3. GitHub 인증 → "Install & Authorize"
4. "기존 Git 리포지토리 가져오기" 선택
5. 빌드 설정:
   - 프로덕션 브랜치: main
   - 루트 디렉터리: landing
   - 빌드 명령: (없음)
6. "저장 및 배포" 클릭
7. ✅ 새 "percept" Pages 프로젝트 생성 및 자동 배포 활성화
```

**URL**: percept-czw.pages.dev (임시)

### 배운 점
1. **GitHub Actions 배포 vs 직접 연동**:
   - Actions: wrangler API 토큰 권한 관리 복잡
   - 직접 연동: GitHub App 설치 후 자동 배포 (간단)

2. **CF Pages 프로젝트 타입**:
   - Manual deployment: Git 연동 불가
   - Deploy from Git: Git 연동 가능

3. **배포 경로 문제**: GitHub repo root = `/Volumes/My Passport for Mac/My_project/perceptdot/`
