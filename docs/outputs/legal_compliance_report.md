## 1. 통신판매업 신고

### 1.1 신고 의무 조건

**판정: 필수**

전자상거래법(「전자상거래 등에서의 소비자보호에 관한 법률」)에 따라 **온라인을 통해 상품·서비스를 반복적으로 판매하는 모든 사업체는 통신판매업 신고가 필수**입니다.

#### Perceptdot의 경우
- 월 구독 형태의 SaaS 서비스 판매
- 온라인 결제(Stripe)를 통한 반복적 거래
- **신고 의무 확실** ✓

#### 신고 면제 조건 (다음 모두 충족 시)
- 개인사업자이면서
- 6개월 거래 횟수 20회 미만 AND
- 6개월 거래 금액 1,200만 원 미만

**Perceptdot 평가**: 구독 서비스 특성상 신고 면제 기준 미충족 가능성 높음 → **신고 필수**

---

### 1.2 신고 방법

**신고 기관**: 정부24(온라인) 또는 사업장 관할 시·군·구청(방문)

#### 정부24 온라인 신고 절차

1. **정부24 접속** → 로그인
2. **민원 검색**: "통신판매업 신고" 입력
3. **신청**: "통신판매업신고-시.군.구" 선택
4. **유의사항 확인** → "다음으로" 클릭
5. **사업자 정보 입력**: 사업자등록증 기재 사항
6. **신고 제출**
7. **신고증 발급**: 1~3일 후 문자 수신 → 정부24에서 출력 (7일 이내)

#### 신고 전 필수 준비물

- ✓ 사업자등록증
- ✓ PG사(Stripe) 가입 완료
- ✓ 구매안전서비스 이용 확인증 (에스크로 서비스, 선택사항)

#### 비용
- 수수료: **무료**
- 통신판매업 등록면허세: **별도 납부** (지역별 상이)

---

### 1.3 사업자등록번호 필요 여부

**필수 요건**

통신판매업 신고 시 사업자등록번호(10자리) 필수 입력.

#### Perceptdot의 경우
- 한국 법인 설립 → 사업자등록증 발급 필요
- 사업자등록번호: 722-60-00889 (현재 K-사주 용, 별도 법인 여부 확인 필요)

**현황**: 사업자등록번호 미확정 → **먼저 한국 법인 및 사업자등록 완료 필수**

---

### 1.4 신고 없이 운영 시 제재

**판정: 심각한 법위반**

#### 처벌 규정 (전자상거래법)

| 위반 항목 | 처벌 내용 |
|----------|---------|
| **미신고/거짓신고** | 3,000만 원 이하 벌금 |
| **미신고 상태 판매** | 영업정지 15일 이상 + 1,000만 원 이하 과태료 |
| **변경신고 미이행** | 100만~500만 원 과태료(누적) |

#### 실질적 영향
- 법적 책임 추가 (소비자 분쟁)
- 신용등급 하락
- 향후 사업 인허가 제한
- 정부 지원금 지급 불가

**결론**: 신고 **반드시 필수** ✓✓✓

## 2. 이용약관 & 개인정보처리방침

### 2.1 이용약관 필수 기재사항

**판정: 필수**

#### 전자상거래법상 필수 항목

웹사이트 초기 화면에 다음 사항을 **명확하게 표시**해야 함:

1. **사업자 정보**
   - 사업자명 (상호)
   - 사업자등록번호
   - 대표자명
   - 주소
   - 전화번호
   - 이메일

2. **약관 & 정책**
   - 이용약관 (전문 링크/다운로드)
   - 개인정보처리방침
   - 환불정책
   - 배송정책

3. **거래 관련 정보**
   - 상품/서비스 설명 및 가격
   - 결제 방법
   - 배송/서비스 이용 조건
   - AS(고객지원) 안내

4. **약관 개정 시**
   - 적용일자 7일 전 공지 필수
   - 소비자 불리한 변경 시 30일 유예 기간

#### Perceptdot의 경우

SaaS 구독 서비스 특성상:

- ✓ **평가 계획**: 약관 필수
- ✓ **결제 구간**: 신용카드, 구독 자동갱신 정책 명시
- ✓ **환불/취소**: 구독 취소 절차, 환불 조건 명시
- ✓ **데이터 삭제**: 계정 삭제 시 데이터 처리 방식

---

### 2.2 개인정보처리방침 필수 항목

**판정: 필수**

한국의 개인정보보호법(PIPA)에 따라 개인정보 수집·이용 시 처리방침 공시는 **의무**.

#### 필수 기재 항목 (2025 기준)

1. **개인정보 수집·이용**
   - 수집하는 개인정보 항목 (이름, 이메일, 결제정보 등)
   - 수집 방법 (가입, 결제 등)
   - 수집 및 이용 목적
   - 보유·이용 기간

2. **민감정보/고유식별정보**
   - 신용카드번호 등 민감정보는 별도 동의 필수
   - 고유식별정보(주민등록번호) 수집 여부 및 목적

3. **제3자 제공**
   - 누구에게 어떤 정보를 제공하는지
   - 제공 목적 및 기간

4. **개인정보책임자(CPO)**
   - 이름, 직급, 연락처
   - 고충처리 담당 부서

5. **정보주체의 권리**
   - 열람/수정 요청 방법
   - 삭제 요청 절차 (GDPR의 '잊혀질 권리')
   - 처리 정지 요청

6. **행태정보(쿠키) 관련**
   - 쿠키 수집 목적
   - 쿠키 종류 (세션/퍼시스턴트)
   - 거부 방법 (브라우저 설정 등)

7. **자동화된 결정**
   - AI 활용 시: 자동화된 의사결정 방식 설명
   - 부작용 및 이의 제기 방법

#### Perceptdot 특화 항목

**SaaS B2B 서비스**이므로 추가로 고려:

- ✓ **API/연동 정보**: 고객이 타사 서비스와 연동 시 데이터 흐름
- ✓ **로그 데이터**: 서비스 사용 로그, GA4 등 분석 도구
- ✓ **구독 정보**: 결제 프로세스, 자동갱신 정보
- ✓ **국제 전송**: 미국 서버(Stripe, Vercel 등)에 데이터 전송 시 별도 동의

## 3. 쿠키 동의 배너

### 3.1 한국 법 기준 (PIPA)

**판정: 필수 (기능적 쿠키는 권고)**

#### 동의 여부별 분류

| 쿠키 유형 | 동의 필수 | 비고 |
|----------|---------|------|
| **세션 쿠키** (기능적) | 권고 | 로그인 상태 유지 등 |
| **트래킹/분석** (GA4) | **필수** | 개인정보보호법상 행태정보 |
| **타겟팅/광고** | **필수** | 별도 사전 동의 필수 |
| **제3자 쿠키** | **필수** | Stripe, Vercel 등 |

#### 핵심 규칙

**한국 개인정보보호법(PIPA)**:
- 쿠키가 개인 식별정보와 결합되면 **개인정보로 간주**
- 행태정보(맞춤형 광고) 수집 시 **사전 동의 필수**
- 동의 거부 방법을 명확히 제시해야 함

**필수 공시 위치**: 개인정보처리방침에 다음 기재
- 쿠키 사용 목적
- 종류 및 보유 기간
- 거부 방법 (브라우저 설정 등)

---

### 3.2 GDPR (유럽) vs 한국 PIPA 비교

#### 주요 차이점

| 항목 | GDPR | PIPA (한국) |
|------|------|-----------|
| **동의 기준** | Opt-in (선택형) | Strict Opt-in (엄격) |
| **쿠키 분류** | 필수/비필수 분리 | 개인정보 여부만 판단 |
| **국제 전송** | 적정성 결정/계약 | **별도 동의** 필수 |
| **처벌** | 매출 4~6% 과징금 | 3,000만 원 이하 벌금 |
| **제재 강도** | 매우 높음 | 비교적 낮음 |

#### 결론
- **GDPR이 더 엄격** (유럽 사용자 우선)
- 한국 PIPA는 **동의 기준은 엄격하되 제재는 약한 편**
- GDPR 준수 시 PIPA도 대부분 충족됨 (역은 아님)

---

### 3.3 영문 랜딩페이지 적용 기준

**판정: 한국 사용자 대상 시 PIPA + GDPR 이중 준수**

#### 시나리오별 가이드

**A) 한국 사용자 + 유럽 사용자 동시 타겟팅**

```
요구사항:
├─ 국가 감지(geo-IP) 후 분기 적용
├─ 한국 사용자: PIPA 준수 + 기본 쿠키 동의
└─ 유럽 사용자: GDPR 준수 + 쿠키 배너(거부 옵션)
```

**B) 글로벌 영문 사이트 (국가 구분 없음)**

```
적용 기준:
1. GDPR 기준 선택 (더 엄격)
   → GDPR 배너 + opt-out 옵션 제공

2. 또는 한국 기준만 (PIPA)
   → 개인정보처리방침에 쿠키 설명
   → 동의 배너 권고 (의무 아님)
```

#### Perceptdot의 경우

**현황**: 글로벌 영문 랜딩 + 한국 법인 구독 서비스

**권장 방안**:

```
1. Geo-IP 감지 구현
   ├─ KR 사용자: PIPA + 쿠키 동의 배너
   └─ EU 사용자: GDPR + 쿠키 배너

2. 개인정보처리방침
   ├─ 국영문 이중 제공
   └─ 각 지역 법 기준 명시

3. GA4 트래킹
   ├─ 필수 쿠키: 동의 불필요
   └─ 분석/맞춤형: 사전 동의
```

## 4. Stripe 결제 시 추가 요건

### 4.1 한국 사업자 + Stripe 조합의 문제점

**판정: 복잡 (여러 법적 경로 필요)**

#### 현황 및 제약

**한국 법인이 Stripe 직접 사용 불가**:

1. **Stripe의 한국 미지원**
   - Stripe는 미국, 호주, 캐나다, 싱가포르 등 특정 국가만 지원
   - **한국 법인 계정 개설 불가** (금융감독 규제)

2. **우회 방안 1: 해외 법인(싱가포르/미국) 설립**
   ```
   한국 법인 (Perceptdot KR)
   ↓ (대금 수취)
   ↑
   싱가포르 법인 (Stripe 계정) → Stripe
   ```

3. **우회 방안 2: 해외 법인 + 한국 판매대행**
   ```
   해외 법인이 한국 고객에게 직접 판매
   → 별도 외국환거래 신고 필요
   ```

---

### 4.2 외국환거래 신고 필요 여부

**판정: 필수 (해외에서 자금 수취 시)**

#### 요건

**한국 사업자가 해외에서 로열티/용역비를 수취할 경우**:
- 한국은행에 **외국환거래 신고** 필수
- 금액: 연 USD 5만 이상 시 필수 신고

#### Perceptdot의 경우

**시나리오**: 한국 법인 + Stripe(미국) 결제 수취

```
수취 프로세스:
1. 고객 (한국) → Stripe 결제 (USD)
2. Stripe → 해외 법인 계정 (USD)
3. 해외 법인 → 한국 법인 본계좌 (송금)
4. 한국은행 외국환거래 신고 필수
```

**필요 서류**:
- 거래 계약서
- 송금 증명 (Swift)
- 신고서 (한국은행 양식)

---

### 4.3 전자금융업 등록 필요 여부

**판정: 불필요 (Stripe 사용 시)**

#### 구분

**전자지급결제대행업(PG) 등록이 필수인 경우**:
- 직접 신용카드/계좌 수취
- 고객 결제 정보 처리
- 정산·송금 자동화

**Stripe 사용 시 (권장)**:
- Stripe가 PG 역할 수행
- 한국 사업자는 **PG 등록 불필요**
- Stripe를 통한 간접 수취

#### Stripe 사용 시 이점

| 항목 | 직접 PG | Stripe |
|------|--------|--------|
| **등록 자본금** | 10억 원 이상 | 불필요 |
| **인적 요건** | 전산 전문가 5명 이상 | 불필요 |
| **시스템 구축** | 자체 보안 시스템 | Stripe 제공 |
| **규제** | 금융감독청 감시 | 상대적으로 경시 |

**결론**: **Stripe 사용이 훨씬 간단** ✓

---

### 4.4 권장 결제 구조

```
Perceptdot 구독 결제 구조 (권장)
│
├─ Option A: 싱가포르 법인 + Stripe (가장 간단)
│  ├─ 싱가포르에 현지 법인 설립
│  ├─ Stripe 계정 오픈 (SG)
│  └─ 한국 고객 직접 판매 가능
│
├─ Option B: 한국 법인 + 한국 PG (Toss, NHN Commerce)
│  ├─ 한국 PG와 계약
│  ├─ 통신판매업 신고
│  └─ 간단하지만 수수료 높음 (3~5%)
│
└─ Option C: 한국 법인 + 해외 법인 + Stripe
   ├─ 복잡한 구조
   ├─ 외국환거래 신고 필수
   └─ 정산 지연 가능성
```
