## PART 3: 혁신성

### 혁신 점수

| 차원 | 점수 | 이유 |
|------|------|------|
| 기술 신규성 | 3/5 | 구성요소는 있음, 조합과 UX가 새로움 |
| 사용 사례 신규성 | **5/5** | "에이전트가 자기 산출물을 시각으로 자가 검수" — 전례 없음 |
| 시장 포지셔닝 신규성 | **4/5** | 원커맨드 MCP 비주얼 QA = 최초 |
| 타이밍 신규성 | **5/5** | MCP 폭발 + 바이브코딩 대중화 = 완벽한 시점 |

### 진짜 차별점 (아무도 안 한 것)

1. **에이전트 자가 검수 루프**: 현재 도구들은 사람이 diff를 본다. perceptdot/eye는 에이전트가 자기 코드의 결과를 보고 스스로 수정한다
2. **자연어 QA 리포트**: "헤더 버튼이 컨테이너 밖으로 12px 삐져나왔습니다" — 에이전트가 즉시 행동 가능
3. **원커맨드 설치**: `claude mcp add --transport http perceptdot https://mcp.perceptdot.com/mcp` — Applitools/Percy 대비 마찰 제로
4. **바이브코더 타겟**: CSS를 모르는 비개발자가 AI로 UI 만들 때 → 시각적 버그를 본인이 못 잡음. 이 시장이 폭발 중

> "AI Is Blind" — 업계에서 이미 문제로 인식 중. 솔루션은 없음.

## PART 5: 리스크

### 🚨 CRITICAL — 아키텍처 블로커 (즉시 해결 필요)

**Cloudflare Workers에서 Playwright 실행 불가.**

- Workers 메모리 한도: 128MB
- Playwright 브라우저 바이너리: ~400MB+
- 결론: 현재 설계대로 구현 불가

**해결책:**

```
기존 (불가):
Cloudflare Workers → Playwright 직접 실행 ❌

신규 (가능):
Cloudflare Workers (MCP 프로토콜 레이어)
    ↓ HTTP 요청
Browserless.io 또는 Browserbase (관리형 브라우저 서비스)
    ↓ 스크린샷 반환
Claude Vision API (분석)
    ↓ 구조화 리포트
Cloudflare KV (베이스라인 저장)
```

추가 비용: ~$29~99/월 (Browserless.io) → Pro 2~5명이 커버
