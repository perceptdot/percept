### 6.2 CPO 최종 권고: "지금 즉시 피벗" (옵션 A)

**근거**:
1. **기회비용 극히 낮다**: 추가 개발 3~4일, 기존 코드 변경 0%. 실패해도 손실 = 3~4일뿐.
2. **대기의 기회비용이 더 크다**: 약한 해자 상태에서 "첫 유저 전 경쟁사 진입" 리스크.
3. **TAM 확장이 생존에 필수**: 솔로 파운더가 초니치에서 생존 어려움.
4. **병렬 실행 가능**: 마케팅(기존 초안) + core 개발 동시 진행.
5. **바이럴 구조 근본 차이**: K=0.1→0.5~1.5. 콜드 스타트를 깨는 열쇠.

### 6.3 실행 순서 권고

```
Week 1 (병렬):
  ├── [마케팅] 기존 초안으로 Twitter/Reddit/HN 포스팅 시작
  ├── [개발] @perceptdot/core Phase 1 개발 (1~2일)
  └── [개발] 백엔드 Registry 프록시 Phase 2 (1일)

Week 2:
  ├── [DevRel] 랜딩/README 업데이트 (0.5일)
  ├── [마케팅] core 포함 메시지로 2차 마케팅
  └── [QA] core 통합 테스트
```

### 6.4 실패 기준 (Pull-back Trigger)

- 8주 후 percept_discover 호출률 < 5%
- 12주 후 무료 유저 < 50명
- 커뮤니티 NPS -30 이하

→ core 패키지만 deprecate, 기존 4개 서버 영향 없음.
