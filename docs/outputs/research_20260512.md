## 2. 무엇이 진짜 새로운가

**기존 코딩 에이전트**: "이 코드 짜줘" → 작업물 반환 (단방향 가치)
**이번 사례**: "돈 벌어와" → 시장 탐색 + 가치 생산 + 외부 정산 회수 (**닫힌 경제 루프**)

핵심 깨짐 지점:
1. **에이전트가 task의 발견자가 됨** — 사람이 task를 하달하지 않음
2. **에이전트가 외부 시스템과 협상** — 메인테이너 코멘트 응대, PR 리뷰 반영
3. **에이전트 명의(또는 위임 명의)로 자금 수령** — 결제 인프라가 에이전트를 1차 주체로 인식

이건 단순 "AI가 일했다"가 아니라 **AI가 마이크로-비즈니스를 운영했다**. 22시간/$16.88은 ROI로는 형편없지만(시간당 $0.77, GPT 비용 차감 시 적자 가능성 큼), **proof-of-concept**으로는 결정적.

## 3. perceptdot 관점 — 시사점 4가지

### (A) B2A2H 가설의 강력한 외부 증거
우리 슬로건: *"에이전트가 추천, 사람은 결제만"*
이번 사례: *"에이전트가 일하고, 에이전트가 수익도 회수"*

→ 우리가 B2A2H를 그리던 시점보다 시장이 **한 칸 더** 갔다. "에이전트가 경제 주체"라는 명제는 더 이상 미래 시제가 아님. 사업계획서/랜딩의 톤을 "곧 옵니다"가 아니라 "**이미 시작됐고 우리는 인프라**"로 바꿔야 함.

### (B) 새로운 ICP 후보 — "수익형 자율 에이전트"
지금까지 우리 ICP: 개발자/CEO가 운용하는 코딩 에이전트.
추가 ICP: **자체 수익을 내야 하는 자율 에이전트** (예: 바운티 헌터 봇, 자동 PR 봇, 자율 SaaS 운영 봇).

이들에게 perceptdot이 줄 수 있는 가치:
- "어디서 돈이 되는 작업을 찾을지" 지각 (HackerOne/GitHub Sponsors/Gitcoin/Replit Bounties API 통합)
- "이 작업의 기대 수익 vs 토큰 비용" ROI 계산 (우리 핵심 역량)
- "결제 회수 시 어디로 보낼지" wallet routing

### (C) 가격 신뢰도 — $19/월은 싸다
에이전트가 22h에 $16.88 만든다는 수치를 보고 나면:
- perceptdot Pro $19/월 = 에이전트 **하루치 수익의 1.13개** 수준
- 에이전트가 자기 비용 자기가 결제하는 시대가 오면 **$19 결정은 1초**
- 가격 페이지에 이 수치를 ROI 비교 그래프로 박을 가치 있음 (단, OpenAI 사례 인용 시 출처 명시)

### (D) 위협 — Codex Security가 인접 시장에 진입
같은 검색에서 발견된 **Codex Security**(GPT-5.4 기반)는 이미:
- 120만 commit 스캔, 10,561건 high-severity 발견 (2026-03)
- GnuPG/GnuTLS/GOGS/PHP/Chromium 등 메이저 OSS에서 취약점 발견
- "Aardvark" (OpenAI agentic security researcher)도 별도 발표됨

→ 보안 도메인은 OpenAI가 직접 먹는 중. 우리가 보안 MCP로 들어가면 정면 충돌.
→ 우리 첫 MCP는 보안이 아닌 **수익·운영 데이터 지각**(GA4/Vercel/Stripe 류)로 가야 한다는 기존 노선이 더 단단해짐.
