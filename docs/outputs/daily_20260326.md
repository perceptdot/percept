## 세션 7 (데스크탑 Claude) — visual_check 401 해결 작업

### 원인 분석
- `claude mcp list` → perceptdot: `https://mcp.perceptdot.com/mcp` — `?api_key=` 없음
- MCP Worker(`mcp.perceptdot.com`)가 `api_key` 쿼리 파라미터로 키를 받아 백엔드에 전달하는 구조
- URL에 키가 없으니 `apiKey=null` → 백엔드 401

### github/vercel과의 차이
- github/vercel: 로컬 Node.js process.env 캐싱 문제 (readEnvKey로 해결)
- visual_check: 원격 CF Worker URL에 api_key 파라미터 누락 — 완전히 다른 문제

### 최종 해결 (rotate_percept_key.py)
- settings.json의 기존 PERCEPT_API_KEY → KV team 플랜 등록
- 노출 키(pd_live_5309...) KV 삭제
- ~/.claude.json + ~/.claude/.mcp.json URL에 `?api_key=KEY` 추가
- Claude Code 재시작 완료

### 생성된 스크립트
- `scripts/rotate_percept_key.py` — 키 교체 원스텝 (KV등록+노출키삭제+URL업데이트)
- `scripts/store_key_direct.py` — Cloudflare REST API로 KV에 키 저장 (wrangler 우회용)

### visual_check vs github/vercel 차이
- github/vercel: 로컬 process.env 캐싱 → readEnvKey() 해결
- visual_check: 원격 CF Worker URL에 api_key 파라미터 누락 → URL 업데이트로 해결
