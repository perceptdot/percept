### 제1원칙 분석 (일론 머스크 방식)

| 가정 | 물리/관성 | 진실 |
|------|----------|------|
| "CF Worker secret set하면 즉시 반영" | 관성 | `--env production` = 별도 Worker. `perceptdot-mcp` ≠ `perceptdot-mcp-production` |
| "같은 CF 계정 Worker간 fetch = 일반 HTTP" | **물리적 제약** | custom domain 경유 시 X-Percept-Key header 손실됨 |
| "MCP 서버와 API 서버는 분리 필수" | 관성 | MCP는 100% 프록시. API Worker에 `/mcp` 내장 가능 |
| "domcontentloaded면 스크린샷 가능" | 관성 | Next.js CSR은 JS 실행 후 렌더링. 대기 필요 |
| "has_issues와 summary는 자동 일치" | 관성 | DOM 없고 AI medium/low만 있으면 불일치 발생 |

### 해결 과정

| # | 시도 | 결과 |
|---|------|------|
| 1 | `set_worker_secret.py` → `perceptdot-mcp` secret 등록 | ❌ 대상 Worker 틀림 |
| 2 | `perceptdot-mcp-production`에 secret 등록 | ❌ 여전히 401 |
| 3 | API Worker에 `/mcp` 엔드포인트 내장 (1홉) → `api.perceptdot.com/mcp` | ✅ curl 테스트 성공 |
| 4 | `~/.claude.json` URL 변경 → Claude Code 재시작 | ❌ Claude Code가 여전히 옛날 서버 사용 |
| 5 | header 방식으로 변경 → 재시작 | ❌ 여전히 옛날 서버 |
| 6 | **전략 전환**: 옛날 MCP Worker 코드에 `api_key: apiKey` body 추가 | ✅ **최종 해결** |
| 7 | JS 렌더링 대기(`waitForFunction`) 추가 | ✅ 블랙스크린 해결 |
| 8 | summary 보정 (finalHasIssues 기반) | ✅ 일관성 확보 |

### 최종 수정 (3파일)

1. **`mcp/src/index.ts`**:
   - `api_key: apiKey`를 body에 추가 (header 손실 우회)
   - 키 우선순위: 유저 제공 > Worker secret (외부 사용자 자기 쿼터 사용)

2. **`api/src/index.ts`** (2건):
   - `/mcp` POST 라우트 내장 — Hono `app.fetch()` 내부 호출 (1홉)
   - `waitForFunction` 3초 — JS 프레임워크 렌더링 대기
   - `finalSummary` 보정 — has_issues=false일 때 clean summary

3. **`~/.claude.json`**: URL `mcp.perceptdot.com/mcp` 유지 (키 불필요, Worker secret 자동)

### 테스트 결과

| 사이트 | 결과 | 시간 | 타일 |
|--------|------|------|------|
| example.com | ✅ No issues | 7.5s | 1 |

### 키 아키텍처 (최종)

```
Claude Code → mcp.perceptdot.com/mcp (MCP Worker)
  ↓ Worker secret(PERCEPT_API_KEY)를 body.api_key로 전달
api.perceptdot.com/v1/eye/check (API Worker)
  ↓ body.api_key ?? header X-Percept-Key로 인증
KV 조회 → 정상 응답

또는 (1홉 직접):
외부 클라이언트 → api.perceptdot.com/mcp?api_key=KEY
  ↓ 내부 app.fetch() → /v1/eye/check 직접 호출
```

### 교훈
- CF Worker-to-Worker custom domain fetch는 header가 손실될 수 있음 → body 전달 필수
- `--env production` 배포 시 Worker 이름이 달라짐 → secret 대상 확인 필수
- Claude Code MCP HTTP transport: URL 설정 변경이 즉시 반영 안 될 수 있음 → 서버 코드 수정이 더 안전
