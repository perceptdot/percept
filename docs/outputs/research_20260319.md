## 1. Composio MCP 서버 구조

### 개요
- **정체**: MCP 게이트웨이 + 툴 통합 플랫폼 (시리즈A $29M)
- **핵심 구조**: 에이전트 ↔ Composio 게이트웨이 ↔ 외부 서비스 (500+ 앱)
- **제공 방식**: 관리형 MCP 서버 (Hosted MCP) + 자체 배포(VPC/온프레미스)

### 서버 구성 요소
```
[에이전트] → [Composio MCP Gateway]
                 ├── Auth 레이어 (OAuth2, API Key, Basic Auth 등)
                 ├── Tool Execution Engine (병렬 실행 지원)
                 ├── Rate Limiter
                 ├── OpenTelemetry 기반 로깅
                 └── RBAC + 감사 로그 (Enterprise 전용)
```

### 기술 특성
- **프로토콜**: MCP 표준 (Anthropic spec) 준수
- **인증**: OAuth 2.1, API Key, Basic Auth, Bearer Token 모두 지원
- **보안**: SOC2 + ISO 인증, 토큰 무저장 아키텍처
- **관측**: OpenTelemetry 기반 — 레이턴시, 에러율, 툴 호출 추적

## 2. Composio가 제공하는 도구 목록과 방식

### 통합 앱 (500+ 종)
| 카테고리 | 예시 |
|---|---|
| 협업 | Slack, Notion, Linear, Trello |
| 생산성 | Gmail, Google Calendar, Google Sheets |
| 개발 | GitHub, Sentry, Datadog |
| CRM | HubSpot, Salesforce, Apollo |
| 소셜 | X(Twitter), WhatsApp, Instagram, TikTok |
| AI 도구 | OpenAI, Anthropic, Perplexity 등 |

### 핵심 내장 툴
| 툴명 | 역할 |
|---|---|
| `COMPOSIO_SEARCH_TOOLS` | 에이전트가 필요한 툴 탐색 |
| `COMPOSIO_MULTI_EXECUTE` | 병렬 툴 실행기 |
| `COMPOSIO_EXECUTE_CODE` | 코드 실행 샌드박스 |
| `COMPOSIO_MANAGE_CONNECTIONS` | 연결 관리 (OAuth 흐름) |
| `COMPOSIO_BASH` | bash 명령 실행 |

### 제공 방식
- **단일 툴킷 MCP**: 특정 앱 하나 (예: `mcp.composio.dev/github`)
- **커스텀 멀티 MCP**: 여러 앱을 하나의 서버에 묶어 배포
- **API 직접 통합**: Python/JS SDK 제공

## 3. ROI 측정 기능 유무

**결론: 없음 (개발자 내부 관측만 존재)**

| 기능 | Composio 여부 | 비고 |
|---|---|---|
| 툴 호출 횟수 로깅 | O | 개발자 콘솔 내부 |
| 레이턴시 추적 | O | OpenTelemetry |
| 토큰 소비량 측정 | △ | MCP Gateway 수준 (집계 없음) |
| 토큰 절감 추정치 | X | 미존재 |
| 비용 절감 계산 | X | 미존재 |
| ROI 대시보드 | X | 미존재 |
| 비즈니스 임팩트 보고 | X | 미존재 |

- 감사 로그는 **Enterprise 최상위 플랜**에만 제공 (락인)
- 수집 데이터는 "개발자/DevOps가 보는 기술 지표"에 국한

## 4. 에이전트 → 오너 보고 기능 유무

**결론: 없음 (구조적 공백)**

Composio의 타겟은 **개발자(에이전트 빌더)** 이다.

```
Composio 현재 플로우:
  에이전트 → Composio 게이트웨이 → 외부 서비스
                    ↓
             개발자 콘솔 (기술 지표)
  ← 오너(CEO/비개발자)에게 도달하는 경로 없음
```

- 에이전트가 오너에게 직접 보고하는 기능: **전무**
- 비개발자가 이해할 수 있는 리포트: **전무**
- 자동 요약/알림 → 슬랙/이메일 발송: **전무**

---

## 5. 토큰 절감 측정 유무

**결론: 없음**

- MCP Gateway 자체는 캐싱 지원 → 반복 요청 중복 제거 가능
- 단, 절감량 수치화/리포팅 없음
- "얼마나 절감됐는가"를 에이전트가 알 수 없음
- 에이전트가 오너에게 "토큰 47% 절감" 메시지 전달 불가

## 6. percept 차별화 포인트 (3개 이상)

### 차별화 포인트 1 — "에이전트 → 오너" 방향 유일

```
Composio: 개발자가 에이전트 설정 → 기술 지표 개발자가 봄
percept:  에이전트가 ROI 측정 → 오너에게 자동 보고 → 오너가 결제
```

- Composio를 포함한 모든 관측 도구(Langfuse, Helicone, Datadog, LangSmith)는
  **개발자 내부용** — 오너 레이어 없음
- percept는 **B2A2H** 구조: 에이전트가 세일즈맨, 오너가 구매자
- 경쟁자 0인 공백 시장

### 차별화 포인트 2 — MCP-native ROI 측정 (툴 단위)

```typescript
// Composio: 없음
// percept: 모든 MCP 서버에 필수 내장
interface PerceptMetrics {
  tool_name: string;
  tokens_saved_estimate: number;   // 툴 호출로 절감된 토큰
  time_saved_ms: number;           // 절감된 시간
  calls_count: number;             // 호출 횟수
  cost_saved_usd: number;          // 환산 비용 절감
}
```

- Composio는 "툴 연결"에 집중, ROI 계산 없음
- percept는 모든 패키지(`@percept/ga4`, `@percept/vercel` 등)에
  토큰 절감 추정치를 내장 — 에이전트가 즉시 참조 가능

### 차별화 포인트 3 — B2A2H 자동 세일즈 플로우

```
에이전트 (Claude Code 등):
"지난 30일 @percept/ga4 사용으로
 토큰 47% 절감 = $38 절약 추정.
 현재 Free 플랜 한도 도달.
 $19/월 업그레이드 → [결제링크]"
         ↓ 오너가 링크 클릭 → 결제
```

- Composio는 개발자가 능동적으로 유료 전환 — 온보딩 복잡
- percept는 에이전트가 ROI 수치를 들고 오너에게 직접 세일즈
- **사람이 이해 안 해도 됨** — 에이전트가 모두 설명

### 차별화 포인트 4 — 외부 서비스 "지각 능력" (Composio는 액션만)

| 관점 | Composio | percept |
|---|---|---|
| 방향 | 에이전트 → 외부 서비스 (쓰기/실행) | 외부 서비스 → 에이전트 (읽기/지각) |
| 철학 | 에이전트에게 **손** 제공 | 에이전트에게 **눈** 제공 |
| 예시 | GitHub PR 생성, Gmail 발송 | GA4 트래픽 지각, Vercel 배포 상태 지각 |

- Composio: 500+ 앱을 **조작**하는 툴
- percept: 외부 서비스 데이터를 **읽고 이해**하는 지각 레이어 → ROI로 환산

### 차별화 포인트 5 — 에이전트 생태계 네이티브 배포

```
Composio: npm 패키지 없음, API 호출 방식
percept:  @percept/* npm 패키지 → 에이전트 README 자동 전파
```

- `npx @percept/ga4` 한 줄 → 에이전트가 즉시 사용
- Claude Code / Cursor / Windsurf 모두 MCP 서버로 자동 인식
- Composio는 "개발자가 설정" 필요 — percept는 "에이전트가 발견"

## 7. 경쟁 포지셔닝 요약

```
[툴 연결 레이어]      Composio    ← 경쟁 안 함
[관측 레이어]         Langfuse / Datadog  ← 개발자용, 경쟁 안 함
[ROI 측정 레이어]     (공백)  ← percept 진입
[에이전트→오너 레이어] (공백)  ← percept 독점
```

- percept는 Composio를 대체하지 않음
- percept는 Composio 위에 올라타는 **ROI 레이어** (보완재)
- 장기적으로 Composio 파트너십 가능성 있음

## 8. 인수인계 (다음 에이전트 참조)

- **CPO/PM**: 차별화 포인트 1~3이 PRD 핵심 — MVP 범위 = 포인트 1+2
- **Dev**: `PerceptMetrics` 인터페이스 확정, 모든 패키지에 필수 삽입
- **Growth**: 차별화 포인트 3(B2A2H 세일즈 플로우) = 핵심 카피라이팅 소재
- **DevRel**: 에이전트용 README에 "Composio와 다른 점" 섹션 추가 필요
